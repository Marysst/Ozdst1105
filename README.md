O'z DSt 1105:2009 — Програмна модель та автоматизований діагностичний стенд (Test Bench)



Цей репозиторій містить інженерну програмну реалізацію мовою Python та комплексне середовище випробувань для національного стандарту симетричного блокового шифрування Узбекистану **O'z DSt 1105:2009**. 



Проєкт розроблено в рамках виконання науково-дослідної роботи (дипломного проєктування та переддипломної практики) для верифікації криптографічних властивостей, перевірки алгебраїчної зворотності та профілювання продуктивності алгоритму.



## Вміст репозиторію

* **Криптографічне ядро:** Повна реалізація математичного апарату шифру в кільці лишків.

* **Автоматизований Test Bench:** Кастомний об'єктно-орієнтований фреймворк на базі `unittest`, який включає 16 діагностичних груп та 99 ізольованих модульних перевірок.

* **Підсистема аналітики:** Вбудований модуль хронометражу мікрооперацій та генерації строгих консольних ASCII-звітів у реальному часі.

* **Документація:** Вихідні тексти звітів та документації.



## Основні можливості та результати

1. **Алгебраїчна коректність (Roundtrip Consistency):** Доведено 100% здатність алгоритму до безвтратної обробки даних у промислових режимах ECB та CBC (із застосуванням PKCS#7).

2. **KAT (Known Answer Tests):** Локалізовано алгоритмічну розбіжність із контрольним прикладом стандарту (PDF-еталоном) на етапі ініціалізації S-боксів, що підтверджує наявність структурних неточностей в офіційній специфікації.

3. **Лавинний ефект (SAC):** Емпірично доведено високу швидкість дифузії — зміна лише 1 біта відкритого тексту або ключа призводить до зміни понад 30% бітів шифртексту вже на перших ітераціях.

4. **Профілювання часу (Bottlenecks):** Визначено, що процедура розгортання ключів (Key Setup) є найбільш ресурсомістким вузлом (~250+ мс), що виконує позитивну функцію «розтягування ключа» (Key Stretching) для захисту від атак повного перебору.



## Як запустити тести

Для запуску діагностичного стенда не потрібні сторонні залежності. Використовується стандартна бібліотека Python 3.10+.



Запуск повного циклу випробувань (99 тестів):

python test_ozdst1105.py
